# Future Source Structure

This directory is intentionally source-code free. Add framework source files here after the implementation plan is approved.

Recommended organization:
- `pages/` for route-level pages
- `layouts/` for shell layouts
- `components/` for reusable UI and content sections
- `content/` for menu and restaurant data
- `styles/` for global styles, tokens, and utilities
- `assets/` for imported assets
- `lib/` for content loading, SEO helpers, and validation utilities
- `types/` for shared type definitions once TypeScript is introduced
