# Components

Future components should be reusable and content-driven.

Suggested component groups:
- `navigation/` - header, mobile menu, footer navigation
- `menu/` - category navigation, menu grid/list, menu item card, dietary badges, price display
- `sections/` - hero, hours/location, featured dishes, specials, about, CTA sections
- `forms/` - contact, newsletter, reservation handoff if needed
- `ui/` - buttons, cards, badges, icons, containers, skip links
