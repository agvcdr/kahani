# Claude Code Workflow

Start each feature by selecting the most relevant skill or agent.

Recommended sequence for first implementation:
1. Use `restaurant-requirements` to capture requirements and missing content.
2. Use `site-architect` to choose framework and component boundaries.
3. Use `menu-data-workflow` and `menu-data-architect` to define the content model.
4. Use `design-responsive-layout` and `responsive-ui-reviewer` for layout planning and review.
5. Use `accessibility-seo-review` and `accessibility-seo-reviewer` before feature completion.
6. Use `cross-browser-test-plan` and `cross-browser-qa` before release.
7. Use `performance-budget` and `performance-reviewer` before deployment.
8. Use `local-testing-workflow` to keep local commands and QA steps accurate.

Direct slash command examples:
- `/restaurant-requirements`
- `/menu-data-workflow`
- `/responsive-layout-review`
- `/accessibility-seo-review`
- `/cross-browser-test-plan`
- `/performance-budget`
- `/local-testing-workflow`
