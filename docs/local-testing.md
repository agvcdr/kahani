# Local Testing Plan

This repository does not include application code yet. Once code is added, define repeatable commands for local testing here.

Expected local checks:
- Install dependencies
- Start the local development server
- Preview a production build locally
- Validate menu content files against the content schema
- Run unit tests for content helpers and formatting utilities
- Run end-to-end tests across key customer flows
- Run accessibility tests
- Run responsive visual snapshots
- Run performance checks

Critical customer flows to test:
- Visitor opens home page on mobile
- Visitor finds menu categories and item details
- Visitor finds hours and location
- Visitor taps call, directions, reservations, or ordering link
- Visitor views menu with unavailable or seasonal items
- Visitor uses keyboard navigation
